## Sprint 1: Kod Temizliği & Yapısal Refactoring

### S1.1 - `__init__.py` Refactoring
- **Problem:** 264 satırlık monolitik export, her şeyi tek yerden açıyor
- **Çözüm:** Lazy import pattern, modül bazlı `__all__` tanımları
- **Hedef:** Her modül kendi public API'sini yönetsin, `__init__.py` < 50 satır

### S1.2 - `main.py` Modernizasyonu
- **Problem:** Frame dosya seçimi her step'te tekrarlanıyor (DRY ihlali)
- **Çözüm:** `_get_analysis_frame()` helper, state machine pattern
- **Hedef:** Pipeline class < 300 satır

### S1.3 - Logger Sistemi
- **Problem:** `print()` tabanlı Logger class, production-grade değil
- **Çözüm:** Python `logging` modülüne geçiş, structured JSON logging
- **Hedef:** Tüm modüllerde tutarlı log format

### S1.4 - Config Management
- **Problem:** Magic numbers ve hardcoded paths dağınık
- **Çözüm:** Merkezi config sistemi (dataclass + TOML/YAML)
- **Hedef:** `config.py` veya `pyproject.toml` ile tek noktadan yönetim

## Sprint 2: Scoring & Analiz Motoru İyileştirme

### S2.1 - Scoring Pipeline v2
- **Problem:** Mevcut bio_score tek boyutlu, ağırlıklar statik
- **Çözüm:** Multi-dimensional scoring, configurable weights, confidence intervals
- **Hedef:** Composite score + uncertainty band

### S2.2 - Pocket Persistence Analysis
- **Problem:** Sadece tek frame analiz ediliyor, temporal bilgi kaybolıyor
- **Çözüm:** Frame-to-frame pocket tracking, persistence score
- **Hedef:** "Bu cep kaç frame'de açık kaldı?" sorusuna cevap

### S2.3 - Comparative Analysis
- **Problem:** Proteinler arası karşılaştırma mekanizması yok
- **Çözüm:** Cross-protein pocket similarity, structural alignment
- **Hedef:** "Bu cebi başka hangi proteinde gördük?" sorusuna cevap

## Sprint 3: Phase 7 Scaffold - AI Signal Layer

### S3.1 - Dataset Builder
- **Problem:** Classifier için etiketli veri seti yok
- **Çözüm:** `src/ml/dataset.py` - feature extraction, label assignment
- **Hedef:** Train/val/test split, leakage kontrolü, manifest dosyası

### S3.2 - Feature Engineering
- **Problem:** Scoring sadece rule-based
- **Çözüm:** `src/ml/features.py` - pocket geometri, kimyasal, dinamik features
- **Hedef:** 20-30 feature vector per pocket

### S3.3 - Baseline Classifier
- **Problem:** ML modeli yok
- **Çözüm:** `src/ml/classifier.py` - sklearn tabanlı baseline (RF, XGBoost)
- **Hedef:** PR-AUC > rule-based baseline

### S3.4 - Evaluation Framework
- **Problem:** Model değerlendirme mekanizması yok
- **Çözüm:** `src/ml/evaluation.py` - metrics, ablation, calibration
- **Hedef:** Reproducible experiment framework

## Sprint 5: Performance & Developer Experience

### S5.1 - Profiling & Hotspot Fix
- **Problem:** Hangi fonksiyonlar yavaş, bilinmiyor
- **Çözüm:** cProfile + line_profiler, bottleneck tespiti
- **Hedef:** %30+ hız artışı

### S5.2 - Caching Layer
- **Problem:** Aynı protein tekrar analiz edildiğinde sıfırdan başlıyor
- **Çözüm:** SQLite + file cache, invalidation policy
- **Hedef:** Tekrarlı analizlerde %90+ hız artışı

### S5.3 - CLI v2
- **Problem:** `argparse` tabanlı basit CLI
- **Çözüm:** `click` veya `typer` ile modern CLI
- **Hedef:** Subcommands (analyze, batch, report, serve)

### S5.4 - Developer Tooling
- **Problem:** pyproject.toml eksik, pre-commit hooks yok
- **Çözüm:** Modern Python packaging (pyproject.toml, ruff, pre-commit)
- **Hedef:** `pip install -e .` çalışsın
